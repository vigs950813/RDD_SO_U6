6.2.3 Paravirtualización

¿Qué es? La paravirtualización es una técnica de virtualización en la que el sistema operativo invitado está modificado para ser consciente del entorno virtual en el que se ejecuta. Esto permite que el sistema invitado colabore directamente con el hipervisor, logrando una mayor eficiencia y menor sobrecarga en comparación con la virtualización completa. La paravirtualización reduce la necesidad de emulación completa de hardware, ya que el sistema invitado es capaz de comunicarse de manera más directa con el hipervisor.

Ventajas y Desventajas:

  • Ventajas:
    • Mayor rendimiento: Al reducir la emulación de hardware, la paravirtualización ofrece un rendimiento más eficiente.
    • Menor sobrecarga: El sistema operativo invitado puede interactuar directamente con el hipervisor, reduciendo la sobrecarga.
    • Mejor gestión de recursos: Permite una distribución de recursos más controlada y eficiente.
  • Desventajas:
    • Requiere modificaciones en el sistema operativo invitado: No todos los sistemas operativos están diseñados para soportar paravirtualización.
    • Limitaciones de compatibilidad: Debido a la necesidad de modificar el sistema operativo, no todos los entornos son compatibles.
    • Dependencia del hipervisor: La paravirtualización está fuertemente vinculada al hipervisor que soporta esta tecnología.

Tipos de Paravirtualización: Existen diferentes enfoques de paravirtualización que varían según la implementación y el nivel de intervención en el sistema operativo invitado. Algunos ejemplos incluyen:

  • Paravirtualización a nivel de kernel: Modificaciones en el núcleo del sistema operativo para mejorar la comunicación con el hipervisor.
  • Paravirtualización parcial: Solo ciertas partes del sistema operativo son adaptadas para interactuar con el hipervisor, manteniendo compatibilidad en otros aspectos.
  • Paravirtualización completa: El sistema operativo está completamente modificado para funcionar en un entorno paravirtualizado, logrando mayor eficiencia.
